# Форматы файлов и экспорт

## Project JSON

Project JSON - редактируемый формат 3D Tools CAD.

Содержит:

- version;
- objects.

Используется для:

- сохранения проекта;
- повторного открытия сцены;
- сохранения параметров объектов;
- сохранения SVG/text/baked mesh данных.

Project JSON не является производственным mesh-форматом для слайсера.

## 3MF

3MF - предпочтительный формат для 3D-печати.

Преимущества:

- хранит единицы измерения;
- лучше подходит для manufacturing workflow;
- компактнее и богаче STL;
- zip-based контейнер.

Текущий export:

- применяет boolean pipeline;
- экспортирует геометрию в миллиметрах;
- создаёт `model.3mf`.

## STL

STL поддерживается для совместимости.

Ограничения:

- не хранит единицы измерения;
- не хранит материалы/метаданные;
- не хранит topology как 3MF;
- может терять смысл manifold-информации.

В 3D Tools CAD STL экспортируется в миллиметровом масштабе.

## OBJ

OBJ поддерживается для mesh workflow и внешних редакторов.

Используй OBJ, если нужно:

- открыть модель в mesh-редакторе;
- проверить сетку;
- передать модель в не-CAD pipeline.

Для печати предпочтительнее 3MF.

## SVG

SVG import превращает 2D контуры в extruded 3D object.

Рекомендации:

- простые замкнутые контуры;
- избегать self-intersections;
- избегать слишком плотных path;
- после импорта проверить printability.

## Экспортный pipeline

```text
CadObject[] -> robust boolean group -> scale to mm -> exporter
```

Для 3MF:

```text
THREE.Mesh -> vertices/indices -> 3D/3dmodel.model -> zip
```
